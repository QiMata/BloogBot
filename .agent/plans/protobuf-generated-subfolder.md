# Execution Plan: Move generated protobuf C# into Models/Generated/

## Goal
Physically separate the five protobuf-generated C# files (~125k lines) from the
one hand-written file in `Exports/BotCommLayer/Models/`, so an agent (or human)
opening the folder cannot confuse generated output with editable code. Closes
audit P1 / recommended-refactor #2 (`docs/agent-readability-audit.md`).

## Current behavior
`Exports/BotCommLayer/Models/` holds five `// <auto-generated> DO NOT EDIT!`
files (`Communication.cs`, `Game.cs`, `Pathfinding.cs`, `Database.cs`,
`Scenedata.cs`) beside the only hand-written file,
`WoWActivitySnapshotExtensions.cs`. `protocsharp.bat` regenerates into this
folder (default `OUTPUT_DIR=..`). The `.editorconfig` rule
`[Exports/BotCommLayer/Models/**.cs]` marks the *entire* folder
`generated_code = true`, which incorrectly also covers the hand-written file.

## Proposed behavior
The five generated files live in `Exports/BotCommLayer/Models/Generated/`.
`WoWActivitySnapshotExtensions.cs` and `README.md` stay in `Models/`.
`protocsharp.bat` regenerates into `Generated/`. The `.editorconfig` rule
targets `Models/Generated/**.cs`, so only generated files are marked generated
and the hand-written file is correctly formatted/flagged like normal code.
Namespaces are unchanged → no consumer code changes.

## Files likely to change
- **Moves (git mv, no content change):** the five generated `*.cs` →
  `Exports/BotCommLayer/Models/Generated/`.
- **Generator:** `Exports/BotCommLayer/Models/ProtoDef/protocsharp.bat` default
  `OUTPUT_DIR` `..` → `..\Generated`. (`protocpp.bat` untouched — it writes
  outside the repo to `../../../../ActivityManager`.)
- **Build config (infra):** `.editorconfig` glob
  `[Exports/BotCommLayer/Models/**.cs]` → `[Exports/BotCommLayer/Models/Generated/**.cs]`.
- **No `.csproj` change:** `BotCommLayer.csproj` is SDK-default-glob (only
  `bin/**` excluded); moved files stay in-project.
- **Docs:** `protobuf.instructions.md` (`applyTo:` front-matter + body),
  `Models/README.md`, `Exports/BotCommLayer/CLAUDE.md` (incl. the `--csharp_out`
  example), `Exports/BotCommLayer/README.md`, `Services/SceneDataService/CLAUDE.md`,
  and `docs/agent-readability-audit.md` (P1 + deferred table).
- **Frozen-area note:** this is **wire-contract-adjacent** (the protobuf
  contract). The contract bytes do not change — only file location + regen
  output path.

## Tests to add/update
None added. Existing IPC/protobuf tests (`Tests/BotRunner.Tests/IPC/`,
`Tests/PathfindingService.Tests/`) exercise the contract and must stay green in
CI. The real proof is `dotnet build` of the solution + a regen no-op diff.

## Compatibility concerns
None at the wire level — message definitions, field numbers, and namespaces are
unchanged. ~14 consuming projects reference the BotCommLayer *assembly*, not the
file paths, so they are unaffected. Only risk is a stale path reference in
config/docs (mitigated by the reference sweep).

## Migration concerns
One-time: contributors must regenerate into `Generated/` going forward
(handled by the updated `protocsharp.bat` default). A half-applied move is
detected by a build error (missing/duplicate type) or a non-empty regen diff.

old `protocsharp.bat`/`.editorconfig`. No data/schema/wire migration to reverse.
Watch CI build + IPC tests after revert to confirm.
