# Hearthstone ‚Äî Item Presence ‚Üí Bind Validation ‚Üí Combat Gate ‚Üí Channel-Interrupt Gate ‚Üí CMSG_USE_ITEM ‚Üí 10s Cast ‚Üí Homebind Teleport ‚Üí 60-Min Cooldown

First Travel sub-family after the NPC family (`quest-giver.md` / `vendor.md` / `trainer.md` / `gossip.md` / `flight-master.md` / `innkeeper.md`). Where flight-master encompasses **paid server-arbitrated spline travel** and mage-teleport encompasses **class-gated free instant travel**, the hearthstone is the **universal-class single-charge cooldown-amortized teleport-to-bind** ‚Äî every bot from L1 starts with one, every class can use it, and the only cost is the 60-minute cooldown amortization.

This is the **simplest of the five travel sub-families** in terms of the protocol surface ‚Äî one CMSG (`CMSG_USE_ITEM = 171 / 0x00AB`, verified at [`Opcodes_1_12_1.h:174`](D:/MaNGOS/source/src/game/Protocol/Opcodes_1_12_1.h)), one server-side spell effect (`SPELL_EFFECT_TELEPORT_UNITS` with `TARGET_LOCATION_CASTER_HOME_BIND` per [`SpellEffects.cpp:2256-2266`](D:/MaNGOS/source/src/game/Spells/SpellEffects.cpp)), and one cooldown application (`AddCooldown(spellInfo=8690, itemProto=6948)` per [`Player.cpp:21380-21382`](D:/MaNGOS/source/src/game/Objects/Player.cpp)). But the **decision-engine surface is large** because Hearth competes with flight, mage-tele, warlock-summon, boat/zep, and land travel ‚Äî and the bind-point setting (R13 of `innkeeper.md`) is an **upstream prerequisite** that the bot must have already completed before any travel-Objective can route through Hearth.

The shipped `UseHearthstoneTask` ([`Exports/BotRunner/Tasks/Travel/UseHearthstoneTask.cs`](../../../Exports/BotRunner/Tasks/Travel/UseHearthstoneTask.cs), 126 LOC) is a **five-state machine** (`FindItem ‚Üí StopAndCast ‚Üí WaitForCast ‚Üí DetectTeleport ‚Üí Complete`) that reads `IObjectManager.Items` for item `6948` presence, dispatches `ObjectManager.CastSpell("Hearthstone")` (the cast-by-name path that resolves to spell `8690` server-side), then polls position-delta / mapId-change for 15s post-cast as the teleport-detection oracle.

## Overview

Hearthstone interaction in 1.12.1 splits into **five phases**, each with distinct gating predicates and bot-side dispatch patterns.

**Phase 1 ‚Äî Item presence + bind validation.** The bot must possess item `6948` in any inventory slot (main inventory or any bag ‚Äî server's `Player::GetItemByPos` iterates all owned slots). Bot-side: `ObjectManager.Items.Any(i => i.ItemId == 6948)` ([`UseHearthstoneTask.cs:54-55`](../../../Exports/BotRunner/Tasks/Travel/UseHearthstoneTask.cs)). Crucially, the **bind point must be set** ‚Äî every character starts with their racial-capital bind (per [`Player.cpp:21584-21589`](D:/MaNGOS/source/src/game/Objects/Player.cpp) ‚Äî Human=Northshire, Orc=Valley of Trials, etc.) but visiting an innkeeper (R13 of [`innkeeper.md`](../npc/innkeeper.md)) flips `m_homebind` to that inn's coord. Bot must have observed at least one `SMSG_PLAYERBOUND` event during the session OR consume the snapshot's `Player.HomebindMapId / HomebindPosition` fields (currently NOT in snapshot ‚Äî open SCo.TRAVEL.HEARTHSTONE.1). Without a valid bind, hearth teleports to the racial-default coord which is rarely the bot's actual quest hub.

**Phase 2 ‚Äî Combat + channel-interrupt gate.** Per [`UseHearthstoneTask.cs:42-49`](../../../Exports/BotRunner/Tasks/Travel/UseHearthstoneTask.cs), if `player.IsInCombat == true` the task pops immediately. Server-side, the 10-second channel breaks on any movement / damage / explicit cancel (channel-interrupt flags `AURA_INTERRUPT_INTERACTING_CANCELS` and movement-flag are set on spell 8690's `AuraInterruptFlags`). Bot stops movement explicitly before cast via `ObjectManager.StopMovement(ControlBits.Front | Back | StrafeLeft | StrafeRight)` ([`UseHearthstoneTask.cs:70`](../../../Exports/BotRunner/Tasks/Travel/UseHearthstoneTask.cs)).

**Phase 3 ‚Äî Cast dispatch.** Bot dispatches `ObjectManager.CastSpell("Hearthstone")` ([`UseHearthstoneTask.cs:73`](../../../Exports/BotRunner/Tasks/Travel/UseHearthstoneTask.cs)) which routes through `SpellcastingManager.CastSpell` ‚Üí `CMSG_CAST_SPELL(spellId=8690)` on BG, or the equivalent FG memory write. The alternative is `CMSG_USE_ITEM(bag, slot, spellId=8690, castCount, itemGuid)` per [`Opcodes_1_12_1.h:174`](D:/MaNGOS/source/src/game/Protocol/Opcodes_1_12_1.h) ‚Äî both paths resolve to the same `Spell::EffectTeleportUnits` server-side. Cast bar lasts **10 seconds** ([`UseHearthstoneTask.cs:31`](../../../Exports/BotRunner/Tasks/Travel/UseHearthstoneTask.cs) `CastDurationMs = 10_000`); during the cast `player.IsCasting == true` and any movement / combat / explicit cancel breaks the channel.

**Phase 4 ‚Äî Server-side teleport + cooldown application.** Once the 10s cast completes, server enters `Spell::EffectTeleportUnits` ([`SpellEffects.cpp:2251-2266`](D:/MaNGOS/source/src/game/Spells/SpellEffects.cpp)) with `EffectImplicitTargetB[effIdx] == TARGET_LOCATION_CASTER_HOME_BIND` (spell 8690's DBC entry). Dispatches `((Player*)unitTarget)->TeleportToHomebind(TELE_TO_SPELL, false)` per [:2264](D:/MaNGOS/source/src/game/Spells/SpellEffects.cpp). The `false` second argument is `hearthCooldown` ‚Äî **counterintuitively, hearthstone CD is NOT applied here** because spell 8690 has `Attributes & SPELL_ATTR_DISABLED_WHILE_ACTIVE` AND its `RecoveryTime` is server-applied independently of the `hearthCooldown` flag (the flag is for the `worldsafelocs.dbc` corpse-teleport path that also calls `TeleportToHomebind`). Once teleport completes, `Player::AddCooldown(*spellInfo, itemProto)` is invoked via the spell's `OnSuccessfulSpellCast` hook with `itemProto=6948` ‚Üí 60-minute cooldown bound to both the spell **and** the item.

**Phase 5 ‚Äî Landing detection.** Per [`UseHearthstoneTask.cs:96-119`](../../../Exports/BotRunner/Tasks/Travel/UseHearthstoneTask.cs), bot polls `player.Position.DistanceTo(_startPosition) > 100f` OR `player.MapId != _startMapId` to detect teleport. Cross-continent binds (Stormwind = mapId 0, Darnassus = mapId 1) catch the mapId change; intra-continent binds (Crossroads = mapId 1, Orgrimmar = mapId 1) catch the distance threshold (typical bind‚Üíquest-hub distance is 200-2000y). Detection timeout `15_000ms` ([`UseHearthstoneTask.cs:32`](../../../Exports/BotRunner/Tasks/Travel/UseHearthstoneTask.cs)) ‚Äî generous buffer over the 10s cast + 1-2s position-snapshot latency.

## Live-test acceptance

A live integration test exercising this Activity must satisfy R8 (clean cross-test state) by:

1. **Server-up assertion** ‚Äî `ServerHealthcheck` returns 200 before the test starts; fail-fast if not (root R4).
2. **Pre-test snapshot predicate** ‚Äî `Player.Health > 0 AND Player.IsInCombat == false AND Player.IsAlive == true AND Player.IsCasting == false AND Player.IsChanneling == false AND (Player.MovementFlags & (MOVEFLAG_FLYING | MOVEFLAG_ONTRANSPORT)) == 0 AND Inventory.Items.Any(i => i.ItemId == 6948)`. Use `.tele` SOAP to position the bot at a known non-bind coord (e.g. opposite-continent capital); use `.modify cooldown -3600000` to clear hearth CD; use `.cast cancel` to clear in-progress channel; use `.unaura` to clear movement-blockers.
3. **Bind-validation predicate** ‚Äî assert `Player.HomebindMapId == ExpectedMapId AND Player.HomebindPosition.DistanceTo(ExpectedCoord) < 100f`. **Bind setup**: use `.tele <inn>` then `.bind` SOAP (admin command) OR push `BindAtInnkeeperTask` via fixture as a test setup step.
4. **Cooldown-gate predicate** ‚Äî assert `Player.GetItemCooldown(6948) == 0` (currently inferable only via `_lastHearthCastTickMs > 3600_000ms` since snapshot field is missing per SCo.TRAVEL.HEARTHSTONE.2).
5. **Cast-dispatch predicate** ‚Äî push `UseHearthstoneTask` via fixture; poll `Player.IsCasting == true` within 1s of push. **Do NOT construct `new ActionMessage { ActionType = ActionType.USE_ITEM }`** in test body ‚Äî that violates [`CLAUDE.md` Test Isolation Rules](../../../CLAUDE.md).
6. **Cast-duration assertion** ‚Äî assert `Player.IsCasting == true` continuously for 9.5-10.5s (10s ¬±5% tolerance); if cast ends before 9.5s, fail with `interrupted` reason.
7. **Teleport-detection predicate** ‚Äî within 15s of cast complete, assert `Player.Position.DistanceTo(_startPosition) > 100f OR Player.MapId != _startMapId`. Tolerance per R10.
8. **Post-teleport position assertion** ‚Äî assert `Player.Position.DistanceTo(HomebindPosition) < 50f` (inn area is typically 30-50y across).
9. **Cooldown-application assertion** ‚Äî post-teleport, assert `_lastHearthCastTickMs` updated AND any subsequent `UseHearthstoneTask` push within 60 minutes pops at `FindItem` with `NotReady` reason (validates R2 cooldown gate).
10. **Disconnect guard** ‚Äî mid-cast disconnect must fail the test (R8); do not paper over with `ReconnectTask`. Mid-flight client disconnect during the 10s cast cancels the spell server-side (channel-interrupt on disconnect).
11. **Screenshot capture** ‚Äî per R11 in root CLAUDE.md: (a) pre-cast (bot stationary, hearthstone icon highlighted); (b) mid-cast (cast bar 50% filled); (c) post-teleport (bot at inn, hearthstone on cooldown). Overwrite `Tests/artifacts/travel/hearthstone/latest-{pre,mid,post}.png`.
