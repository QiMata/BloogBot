# WoW 1.12.1 Leveling Guide — DecisionEngine Knowledge Base

> **Target patch:** WoW **1.12.1 "Drums of War"** (September 2006), the final pre-TBC vanilla patch.
> **Reference server:** VMaNGOS / MaNGOS-Zero.
> **NOT** retail. **NOT** WoW Classic 2019 re-release. **NOT** Classic Era.
>
> When a wiki page mixes vanilla / TBC / Classic re-release info, only the live-2006 1.12.1 facts are authoritative here. Re-release deltas are noted inline but never override the 2006 baseline.

This folder is a structured knowledge base consumed by the **WWoW StateManager DecisionEngine** to choose the next action for a bot from level 1 through full lvl-60 raid-readiness. Every page is built so a code path of the form `engine.PickNextAction(snapshot)` can answer:

> *Given a `WoWActivitySnapshot`, what should this bot be doing in the next 500 ms tick?*

## End-state goals (per character / per account)

The DecisionEngine's terminal predicate is:

1. **All 9 classes leveled to 60** (one per class; faction-locked classes leveled on the matching faction's account)
2. **All primary professions maxed** (300 in each) on at least one alt across the account
3. **Major reputations Exalted** on at least one main per faction-side
4. **All attunement keys** on the main raider(s)
5. **Highest realistic PvP rank** on the main PvP toon (Rank 14 = Grand Marshal / High Warlord; weapons = terminal goal)
6. **Pre-raid BiS or better gear** on every lvl-60 character flagged as a raider

## Conventions for content files

Every new content file in passes 2+ ends with two sections (in this order):

```markdown
## Decision-Engine Rules
- (Bullet list of rules of the form: IF <snapshot predicate> THEN <action> WITH PRIORITY <weight>)

## Snapshot Fields Needed
- (Bullet list of `WoWActivitySnapshot` fields the rules above read; flag any that don't exist yet)
```

These two sections are how the DecisionEngine actually loads the file. Prefer **tables** over prose. Mark thin / unverified sections **`[STUB - pass N]`** so a future iteration can find them.

## Cross-cutting load-bearing rules

- **R1 — No blind sequences/counters/timers.** Gate every action on a snapshot field that proves the state. (See repo root [CLAUDE.md](../../CLAUDE.md) §R1.)
- **VMaNGOS reality check.** Where 1.12.1 retail and VMaNGOS diverge (e.g. Naxx40 / AQ40 boss-script completeness), the file marks the divergence and the engine should consult a server-capability flag.
- **Faction lock.** Paladin = Alliance only. Shaman = Horde only. The engine must never schedule cross-faction class quests.
