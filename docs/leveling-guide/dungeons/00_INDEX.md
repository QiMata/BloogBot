## Standard sections per dungeon file (binding contract)

Every file in this family must contain:

1. **YAML frontmatter** — `title`, `patch`, `sources_crawled` (with mangos.sql line numbers), `crawl_date`.
2. **Quick Facts** table — `map_id`, parent map, linked zone, group size, level range, faction, meeting stone, boss count. Every value cited inline.
3. **Geography & Sub-Zones** — entrance coords + named sub-zones (with AreaTable.dbc TBDs flagged).
4. **Mob Ecology** — creature_template entries with HP, level, EventAI flag.
5. **Boss Table** — entry ID, level, HP, spells (with `creature_template.spell1/2` cited), notable mechanic, source row.
6. **Quest Table** — quest_template entries with quest ID, giver, turn-in, zone, source row.
7. **Recommended Pull Order & Route** — linear or branching, with CC/interrupt callouts.
8. **Decision-Engine Rules** — 15-20 rows in `id | bracket | precondition | action | priority` format.
9. **Snapshot Fields Needed** — code block listing required snapshot fields.
10. **TBD / Backlog table** — items requiring deeper extraction (AreaTable.dbc sub-zones, etc.).
11. **Cross-References** — to social/, combat/, recovery/, npc/, travel/, sections/, decision-engine/.
12. **VMaNGOS / Server Reality Check** — EventAI vs scripted_creature distinction; known divergences.

`ragefire-chasm.md` is the **template authority** — when authoring subsequent dungeon files, mirror its section structure exactly.
