# Melee Rotation — Target Acquisition → Chase → Face → `CMSG_ATTACKSWING` + Cooldown Ladder + Threat Governance

Foundational combat cycle. Every melee Activity (questing, dungeon trash, raid trash, BG mid-fights, world bosses) assumes the bot can pick the right hostile, close the gap to MaNGOS melee range, hold facing without tripping `BADFACING`, drive auto-swings, and weave the per-spec cooldown ladder while obeying party threat policy. This doc covers the generic **melee** path — class/spec specifics live in per-class docs (under `Westworld of Warcraft/BotProfiles/<Spec>/Tasks/PvERotationTask.cs` for the rotation body; this doc is the Activity-level contract every spec implements against).

The shipped runtime split is: target acquisition + chase + facing + `CMSG_ATTACKSWING` live in the shared `CombatRotationTask` ([`Exports/BotRunner/Tasks/CombatRotationTask.cs`](../../../Exports/BotRunner/Tasks/CombatRotationTask.cs)) and atomic `StartAttackTask` / `StopAttackTask` / `CastSpellTask` ([`Exports/BotRunner/Tasks/StartAttackTask.cs:13`](../../../Exports/BotRunner/Tasks/StartAttackTask.cs), [`Exports/BotRunner/Tasks/StopAttackTask.cs:13`](../../../Exports/BotRunner/Tasks/StopAttackTask.cs), [`Exports/BotRunner/Tasks/CastSpellTask.cs:16`](../../../Exports/BotRunner/Tasks/CastSpellTask.cs)). The cooldown ladder + resource gating + AoE-vs-single-target decision live in each per-profile `PvERotationTask` (27 profiles per [Plan/Activities/combat.md](../../Plan/Activities/combat.md)); the canonical example used throughout this doc is `WarriorFury.Tasks.PvERotationTask` at [`BotProfiles/WarriorFury/Tasks/PvERotationTask.cs`](../../../BotProfiles/WarriorFury/Tasks/PvERotationTask.cs). Threat estimation (no server-side threat API in 1.12.1) lives in [`Exports/BotRunner/Combat/ThreatTracker.cs`](../../../Exports/BotRunner/Combat/ThreatTracker.cs).

The server-side state machine spans three handlers: `WorldSession::HandleAttackSwingOpcode` at [`D:/MaNGOS/source/src/game/Handlers/CombatHandler.cpp:32`](D:/MaNGOS/source/src/game/Handlers/CombatHandler.cpp) (drives `Player::Attack` after friend/spawning/dead-target gates at lines 49 + 56), `WorldSession::HandleAttackStopOpcode` at [`CombatHandler.cpp:67`](D:/MaNGOS/source/src/game/Handlers/CombatHandler.cpp), and `WorldSession::HandleSetSelectionOpcode` at [`D:/MaNGOS/source/src/game/Handlers/MiscHandler.cpp:442`](D:/MaNGOS/source/src/game/Handlers/MiscHandler.cpp) (clears combo points for Rogue/Druid on selection change at line 458, cancels auto-shoot on hostile-target-loss at lines 462-475). Threat is accrued via `ThreatCalcHelper::CalcThreat` at [`D:/MaNGOS/source/src/game/Threat/ThreatManager.cpp:35`](D:/MaNGOS/source/src/game/Threat/ThreatManager.cpp) and the most-hated victim queried via `ThreatManager::getMostHated` at [`D:/MaNGOS/source/src/game/Threat/ThreatManager.h:169`](D:/MaNGOS/source/src/game/Threat/ThreatManager.h); per-spell threat is loaded from the `spell_threat` DB row via `SpellMgr::GetSpellThreatEntry` at [`D:/MaNGOS/source/src/game/Spells/SpellMgr.h:430`](D:/MaNGOS/source/src/game/Spells/SpellMgr.h) (loaded at [`SpellMgr.cpp:839`](D:/MaNGOS/source/src/game/Spells/SpellMgr.cpp)).

The melee range formula is `max(NOMINAL_MELEE_RANGE, attacker.CombatReach + target.CombatReach + BASE_MELEERANGE_OFFSET + leeway_if_both_moving)` — encoded in `CombatDistance.GetMeleeAttackRange` at [`Exports/GameData.Core/Models/CombatDistance.cs:43`](../../../Exports/GameData.Core/Models/CombatDistance.cs); MaNGOS constants `ATTACK_DISTANCE = 5.0f`, `NOMINAL_MELEE_RANGE = 5.0f`, `MELEE_RANGE = NOMINAL_MELEE_RANGE - MIN_MELEE_REACH * 2 = 1.0f`, `DEFAULT_COMBAT_REACH = 1.5f`, `MIN_MELEE_REACH = 2.0f`, `LEEWAY_MIN_MOVE_SPEED = 4.97f`, `LEEWAY_BONUS_RANGE = 2.66f` at [`D:/MaNGOS/source/src/game/Objects/ObjectDefines.h:25-57`](D:/MaNGOS/source/src/game/Objects/ObjectDefines.h).

## Overview

The per-tick body in `CombatRotationTask.Update(attackDistance)` at [`Exports/BotRunner/Tasks/CombatRotationTask.cs:79`](../../../Exports/BotRunner/Tasks/CombatRotationTask.cs) drives the generic part: (1) early `TryUseHealthPotion` / `TryUseManaPotion` at lines 82-83 (potions share a 2-min CD per `Config.PotionCooldownMs` at [`CombatRotationTask.cs:501`](../../../Exports/BotRunner/Tasks/CombatRotationTask.cs)); (2) `ObjectManager.GetTarget(Player)` lookup at line 86; (3) `meleeRange = GetMeleeRange(target)` computed from `CombatReach` per [`CombatRotationTask.cs:196-209`](../../../Exports/BotRunner/Tasks/CombatRotationTask.cs); (4) BG sentinel checks `HadRecentMeleeRangeRejection` / `HadRecentMeleeFacingRejection` at lines 98 + 106 (server `BADRANGE` / `BADFACING` reject windows surfaced through `SpellcastingManager` at [`Exports/WoWSharpClient/SpellcastingManager.cs:30-35`](../../../Exports/WoWSharpClient/SpellcastingManager.cs) — `RecentMeleeRejectWindowTicks = 1200ms`); (5) chase via `ObjectManager.Face(target.Position) + TryNavigateToward(target.Position)` if `distance > attackDistance` (line 108); (6) one-tick facing prime on BADFACING (lines 118-138); (7) melee-engage delay until grounded (lines 140-146 — guards against airborne `CMSG_ATTACKSWING` while `MOVEFLAG_FALLINGFAR | MOVEFLAG_JUMPING` set per `IsPlayerAirborne` at line 157); (8) `ObjectManager.Face(target.Position) + ObjectManager.StartMeleeAttack()` (lines 152-153). On the BG side `StartMeleeAttack` ultimately sends `CMSG_ATTACKSWING` at [`SpellcastingManager.cs:386`](../../../Exports/WoWSharpClient/SpellcastingManager.cs).

`EnsureTarget()` at [`CombatRotationTask.cs:352`](../../../Exports/BotRunner/Tasks/CombatRotationTask.cs) drives target acquisition and combat-timeout: it pops the task when `Aggressors.Any() == false` (line 355), counts `_combatDurationTicks` and disengages after `150 ticks ≈ 15s at 100ms tick rate` (lines 364-371 — prevents permanent combat locks against GM-immune mobs), and re-targets via `AssignDPSTarget` (lowest-HP-first) at line 391. Group play uses `AssignGroupTarget` at line 607 which prefers Skull mark → Cross mark → tank's target → lowest HP (`GetMarkedTarget` at line 554 + `GetAssistTarget` at line 579 + `IObjectManager.SkullTargetGuid` / `CrossTargetGuid` at [`Exports/GameData.Core/Interfaces/IObjectManager.cs`](../../../Exports/GameData.Core/Interfaces/IObjectManager.cs)).

The cooldown ladder is per-spec. The Warrior-Fury reference at [`BotProfiles/WarriorFury/Tasks/PvERotationTask.cs:94`](../../../BotProfiles/WarriorFury/Tasks/PvERotationTask.cs) (`ExecuteRotation`) is structured as: (a) **stance / interrupt prelude** — `BerserkerStance` swap at line 100 gated on level + creature-type + target-HP; `Pummel` interrupt at line 102 when target is `IsCasting || IsChanneling`; (b) **resource/CD-gated openers** — `DeathWish` at line 106 (10 rage, target-HP > 80%); `BattleShout` at line 108 (10 rage, no existing buff); `Bloodrage` at line 112 (target-HP > 50%); (c) **execute window** — `Execute` at lines 114 + 149 (15 rage, target-HP < 20%); (d) **AoE branch** — `if (ObjectManager.Aggressors.Count() >= 2)` at line 121: `IntimidatingShout` + `DemoralizingShout` + `Cleave` (20 rage, FacingAllTargets at line 128) + `Whirlwind` (25 rage, BerserkerStance + AggressorsInMelee at line 130); (e) **single-target branch** — `Slam` (15 rage, slamReady callback at line 139), `Bloodthirst` (30 rage), `Hamstring` (10 rage, humanoid + no debuff at line 145), `HeroicStrike` (15 rage @ < 30, 45 @ ≥ 30, on-next-swing at line 147), `SunderArmor` (15 rage, target-HP < 80 + no existing debuff at line 151). On-next-swing abilities (`HeroicStrike`, `Cleave`, `Maul`, `RaptorStrike`) are throttled at `OnNextSwingQueueThrottleMs = 1700ms` per [`CombatRotationTask.cs:52`](../../../Exports/BotRunner/Tasks/CombatRotationTask.cs); self-cast abilities (shouts, `Berserker*`, `Bloodrage`, `LastStand`, etc.) retry-throttled at `SelfCastRetryThrottleMs = 1500ms` per line 53.

Threat governance is **estimated client-side** (1.12.1 has no `SMSG_THREAT_UPDATE`). `ThreatTracker.RecordDamage` at [`Exports/BotRunner/Combat/ThreatTracker.cs:21`](../../../Exports/BotRunner/Combat/ThreatTracker.cs) multiplies by `TANKING_STANCE_MULTIPLIER = 1.3f` (Defensive Stance line 18) and `DAMAGE_THREAT_MULTIPLIER = 1.0f` (line 17). `ThreatTracker.ShouldThrottle` at line 53 uses **110% pull-aggro / 0.9 safety margin** — DPS throttle when own threat ≥ 90% of tank threat. This is the same 110%-in-melee / 130%-at-range pull threshold the MaNGOS server uses in `ThreatManager::selectNextVictim` at [`D:/MaNGOS/source/src/game/Threat/ThreatManager.cpp`](D:/MaNGOS/source/src/game/Threat/ThreatManager.cpp); the server side reads from `SpellMgr::GetSpellThreatEntry` at [`SpellMgr.h:430`](D:/MaNGOS/source/src/game/Spells/SpellMgr.h) using the `spell_threat` DB rows loaded at [`SpellMgr.cpp:839`](D:/MaNGOS/source/src/game/Spells/SpellMgr.cpp) — `SELECT entry, Threat, multiplier, ap_bonus FROM spell_threat WHERE build BETWEEN build_min AND build_max`.
