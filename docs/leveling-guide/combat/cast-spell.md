# Cast Spell — Target Acquisition → Cast-Range / LOS / Resource / Facing Gates → `CMSG_CAST_SPELL` + Cast-Time / Channel Maintenance + Pushback / Interrupt Handling

Foundational generic spell-cast cycle. Every ranged or cast-time-bearing Activity (Mage Frostbolt grinds, Hunter Aimed Shot pulls, Priest Smite, Warlock Shadow Bolt, Druid Wrath, Shaman Lightning Bolt, ranged interrupts, group heals, conjures, buffs, debuffs) assumes the bot can pick a valid target, satisfy `Spell::CheckCast` server-side gates (resource, range, line-of-sight, facing, school lockout, movement restrictions, target validity), drive `CMSG_CAST_SPELL` with a correctly framed `SpellCastTargets` payload, and keep the cast alive across pushback / channel ticks / movement-cancel without leaking GCDs. This doc covers the generic **cast-spell** path used by every per-spec `PvERotationTask` / `PvPRotationTask` / `HealTask` / `BuffTask` / `PullTargetTask` / `ConjureItemsTask` / `SummonPetTask`; melee swings live in the sibling [melee-rotation.md](./melee-rotation.md).

## Overview

The per-tick cast flow inside a `CombatRotationTask`-derived spec rotation is: (1) **selection** — iterate the spec's priority list in `ExecuteRotation` (e.g. `BotProfiles/WarriorFury/Tasks/PvERotationTask.cs:94-154`); each candidate emits a gated `TryCastSpell(...)` / `TryUseAbility(...)` / `TryUseAbilityById(...)` / `TryCastHeal(...)`. First gate that satisfies fires `ObjectManager.CastSpell(spellName, castOnSelf)` and the helper returns `true`, ending the rotation tick. (2) **client-side gates** — `TryCastSpell` at [`CombatRotationTask.cs:260-276`](../../../Exports/BotRunner/Tasks/CombatRotationTask.cs) checks: `condition` (line 262), `target != null || castOnSelf` (line 264-265), `distance ∈ [minRange, maxRange]` (line 267-268), `ObjectManager.IsSpellReady(spellName)` (line 270), and `CanAttemptSelfCast(spellName, castOnSelf)` (line 271). `TryUseAbility` at [`:287-299`](../../../Exports/BotRunner/Tasks/CombatRotationTask.cs) adds the `Energy < cost && Rage < cost` resource gate (line 290-291) and the `CanQueueOnNextSwingAbility` on-next-swing throttle (line 294 — `OnNextSwingQueueThrottleMs = 1700ms` per `CombatRotationTask.cs:52`). (3) **dispatch** — `ObjectManager.CastSpell(spellName, castOnSelf)` at [`SpellcastingManager.cs:190-202`](../../../Exports/WoWSharpClient/SpellcastingManager.cs) resolves `spellName` to the highest-known-rank `spellId` via `GameData.Core.Constants.SpellData.GetHighestKnownRank` (line 193) and forwards to `CastSpell(int spellId, ...)` at [`:204-239`](../../../Exports/WoWSharpClient/SpellcastingManager.cs) which frames the `CMSG_CAST_SPELL` payload (`uint32 spellId` + `uint16 targetFlags` + optional packed target GUID) and sends via `WoWClient.SendMSGPackedAsync(Opcode.CMSG_CAST_SPELL, payload)`. (4) **server resolution** — `HandleCastSpellOpcode` runs validity gates, then `Spell::prepare` runs `CheckCast(true)` and arms the cast-time timer. (5) **per-tick maintenance** — `Spell::update` runs movement-cancel (R6), channel-tick (R10), pushback (R7), interrupt (R8) handlers until `SPELL_STATE_FINISHED`.

`CastSpellTask` is the atomic single-cast variant ([`CastSpellTask.cs:12-39`](../../../Exports/BotRunner/Tasks/CastSpellTask.cs)). One-tick body: (a) `MountUsageGuard.TryGetBlockedReasonForSpell(ObjectManager, spellId, out blockReason)` — if blocked (e.g. mount in indoor / combat / non-flyable zone), log `[MOUNT-BLOCK] spell={spellId}` to diagnostic stream and `Pop()` without dispatch (lines 20-26); (b) `if (targetGuid != 0) ObjectManager.SetTarget(targetGuid)` (line 28-29) — which sends `CMSG_SET_SELECTION` at [`SpellcastingManager.cs:307-328`](../../../Exports/WoWSharpClient/SpellcastingManager.cs) and clears prior melee-rejection sentinels on target switch (line 310-314); (c) `ObjectManager.CastSpell(spellId, castOnSelf)` — sends `CMSG_CAST_SPELL` (line 31); (d) `Pop()` (line 34). This task is dispatched by `ObjectiveType.CastSpell` from the StateManager (`Plan/Activities/combat.md` §PvERotationTask line 8 — `CastSpell` row in the ObjectiveType table). The arena pet-summon test `PetManagementTests::Pet_SummonAndManage_StanceFeedAbility` ([`Tests/BotRunner.Tests/LiveValidation/PetManagementTests.cs`](../../../Tests/BotRunner.Tests/LiveValidation/PetManagementTests.cs)) exercises the `CastSpellTask` atomic path with `CallPetSpellId = 883` and `DismissPetSpellId = 2641`.

`CMSG_CAST_SPELL` payload layouts (all three forms send via the same opcode but the `SpellCastTargets` body diverges by `targetFlags`):

| Form | `targetFlags` | Payload body | Source |
|---|---|---|---|
| **Self / no-target** | `0x0000` (`TARGET_FLAG_SELF`) | `uint32 spellId` + `uint16 0x0000` | `SpellcastingManager.cs:214-219` |
| **Unit target** | `0x0002` (`TARGET_FLAG_UNIT`) | `uint32 spellId` + `uint16 0x0002` + `packedGuid CurrentTargetGuid` | `SpellcastingManager.cs:221-226` |
| **Ground / AoE-at-location** | `0x0040` (`TARGET_FLAG_DEST_LOCATION`) | `uint32 spellId` + `uint16 0x0040` + `float x, y, z` | `SpellcastingManager.cs:241-264` |
| **GameObject target** (lock-picking, herb-bonk-and-cast targeting GO) | `0x0800` (`TARGET_FLAG_OBJECT`) | `uint32 spellId` + `uint16 0x0800` + `packedGuid gameObjectGuid` | `SpellcastingManager.cs:266-280` |

Fishing spells are forced through the no-target path regardless of `CurrentTargetGuid` because the cast lands a bobber, not a unit/dest — fishing IDs `[7620, 7731, 7732, 18248, 33095]` at [`SpellcastingManager.cs:28`](../../../Exports/WoWSharpClient/SpellcastingManager.cs) trigger the `forceNoTarget = isFishingSpell` branch at line 209.

## Live-test acceptance

A live integration test exercising spell-cast must satisfy R8 (clean cross-test state per root CLAUDE.md) by:

1. **Server-up assertion** — `ServerHealthcheck` returns 200 before the test starts; if not, FAIL FAST (root CLAUDE.md R4).
2. **Pre-test snapshot predicate** — both bots strict-alive AND `Player.IsCasting == false` AND `Player.IsChanneling == false` AND `Player.IsInCombat == false`. Test staging hydrates spellbook (`SMSG_INITIAL_SPELLS` consumed) before first rotation tick. `.gm off` required (per WWoW MEMORY: GM mode corrupts UnitReaction bits and breaks combat-bearing spell tests).
3. **Target acquisition predicate** — for combat-cast scenarios, find a level-appropriate target visible in `NearbyUnits` within 20 s of staging. For self-cast scenarios (buffs, conjures, summons), no target needed.
4. **Cast dispatch** — `_bot.SendActionAsync(account, new ObjectiveMessage { ObjectiveType = ObjectiveType.CastSpell, IntParam = spellId, LongParam = unchecked((long)targetGuid), BoolParam = castOnSelf })` → routes to `CastSpellTask(spellId, targetGuid, castOnSelf)`. Assert `ResponseResult.Success` on dispatch.
5. **Mount-block predicate** (if applicable) — for `ObjectiveType.CastSpell` against a mount spell ID while indoor / in-combat / swimming, assert diagnostic `[MOUNT-BLOCK]` emitted AND no `CMSG_CAST_SPELL` packet sent AND task popped within 1 tick.
6. **Range / LOS predicate** — for ranged-cast scenarios, position bot at known good range with LOS; poll `Player.Position.DistanceTo(target.Position) <= maxRange - 1.0` within 5 s. Verify cast dispatched on satisfying tick. For LOS test: position bot behind a known terrain occluder; assert `SMSG_CAST_RESULT == SPELL_FAILED_LINE_OF_SIGHT` from server; assert rotation does not retry the same spell on the next tick without reposition.
7. **Cast-time predicate** — for non-instant casts (Frostbolt 2.5s, Heal 3.0s), poll `Player.IsCasting == true` within 200 ms of dispatch; poll `Player.IsCasting == false` AND target HP/aura delta within `castTime + 500 ms`. Verify `MovementFlags == MOVEFLAG_NONE` for the duration.
8. **Movement-cancel predicate** — initiate a 2.5s cast, then 1s in push a `MoveToward` action moving > 0.5 yd; assert `SMSG_CAST_RESULT == SPELL_FAILED_MOVING` AND `Player.IsCasting == false` immediately after.
9. **Channel predicate** (Mind Flay / Drain Life / Hurricane) — assert `Player.IsChanneling == true` AND `Player.ChannelingId == expectedSpellId` for the duration; assert per-tick aura/damage events match expected tick count.
10. **Pushback predicate** — channel a multi-tick spell while a friendly bot deals damage to the caster; assert `Spell.m_timer` shaves time off (telemetry via `SMSG_CHANNEL_UPDATE`) AND total ticks land < expected on damage windows.
11. **Result classification predicate** — for each `SpellCastResult` in `[OUT_OF_RANGE, NO_POWER, LINE_OF_SIGHT, INTERRUPTED, MOVING, NOT_BEHIND, UNIT_NOT_INFRONT, TARGETS_DEAD, SPELL_IN_PROGRESS, NOT_READY, BAD_TARGETS]`, drive the scenario that elicits it and assert `SMSG_CAST_RESULT` payload matches.
12. **Spell-not-known guard** — attempt `CastSpellTask(spellId: 99999)` (unknown spell) AND assert server-side log line emitted AND bot's `CMSG_CAST_SPELL` still sent (this is the kick-risk path — assertion verifies the guard is **not** firing client-side and the bot relies on server kick).
13. **Disconnect guard** — mid-test client disconnect (per `WorldSocket.cpp:91`) must fail the test (R8) — do not paper over with `ReconnectTask` inside the cast-spell test (separate Activity, see [../recovery/reconnect.md](../recovery/reconnect.md)).
14. **Screenshot capture** — per R11 in root CLAUDE.md, capture FG client screenshots at: (a) pre-cast (target acquired, `IsCasting == false`), (b) mid-cast (cast bar visible, `IsCasting == true`), (c) post-cast (target HP/aura delta visible, `IsCasting == false`), (d) on failure (`SMSG_CAST_RESULT` error-toast visible). Overwrite `Tests/artifacts/combat/cast-spell/latest-{precast,midcast,postcast,error}.png`.
15. **Final state dump** — write `Snapshot.player` (HP/Mana/IsCasting/IsChanneling/ChannelingId/MovementFlags/Auras) + last 20 `WoWActivitySnapshot` deltas (target HP curve + aura applications) + last 5 `SMSG_CAST_RESULT` payloads to `Tests/artifacts/combat/cast-spell/snapshot-final.json` for agent post-mortem.
