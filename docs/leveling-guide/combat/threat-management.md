# Threat Management — Threat-Gen Scaling → Tank-Hold → DPS Throttle → Pickup-Loose-Add → Healer-Aggro Mitigation → Threat Reset

Group-aware combat sub-cycle. Where [melee-rotation.md](./melee-rotation.md) drives the single bot's swing/cooldown ladder, [cast-spell.md](./cast-spell.md) drives any single cast, and [pull-target.md](./pull-target.md) drives the pre-engage handshake, **threat-management** is the cross-bot policy that decides *who* the mob is currently chasing and *how much* damage/healing each party member is allowed to pump before the mob flips victim. This doc picks up exactly where [pull-target.md R14](./pull-target.md) defers: group-aware pull priority, tank-hold strategy, and pickup-loose-add — and it picks up where [melee-rotation.md R10](./melee-rotation.md) leaves off: the 110%-melee / 130%-ranged pull-aggro rule, the DPS throttle, and the threat ceiling.

The shipped runtime split is: client-side estimator at [`Exports/BotRunner/Combat/ThreatTracker.cs`](../../../Exports/BotRunner/Combat/ThreatTracker.cs) (a concurrent `Dictionary<ulong,float>` keyed by attacker GUID; no `SMSG_THREAT_UPDATE` exists in 1.12.1, so the bot estimates from observed combat-log events); loose-add detection at [`Exports/BotRunner/Tasks/CombatRotationTask.cs:712-719`](../../../Exports/BotRunner/Tasks/CombatRotationTask.cs) (`FindLooseAdd` — aggressors whose `TargetGuid != player.Guid`); per-tank threat-rotation bodies at [`BotProfiles/WarriorProtection/Tasks/PvERotationTask.cs`](../../../BotProfiles/WarriorProtection/Tasks/PvERotationTask.cs), [`BotProfiles/PaladinProtection/Tasks/PvERotationTask.cs`](../../../BotProfiles/PaladinProtection/Tasks/PvERotationTask.cs), [`BotProfiles/DruidFeral/Tasks/PvERotationTask.cs`](../../../BotProfiles/DruidFeral/Tasks/PvERotationTask.cs); DPS-side throttle gate at [`CombatRotationTask.cs:660-676`](../../../Exports/BotRunner/Tasks/CombatRotationTask.cs) (`ShouldWaitForThreat` — pause DPS while target is `TargetGuid == 0` waiting for tank to grab).

The server-side state machine is owned by `ThreatManager` per creature. `addThreat` at [`D:/MaNGOS/source/src/game/Threat/ThreatManager.cpp:391-424`](D:/MaNGOS/source/src/game/Threat/ThreatManager.cpp) is the single ingress: it gates GMs (line 403), dead aggressor/dead target (line 407), CC-on-self assist-threat suppression (lines 414-421), then calls `ThreatCalcHelper::CalcThreat` at [`ThreatManager.cpp:35-52`](D:/MaNGOS/source/src/game/Threat/ThreatManager.cpp) which applies `SPELLMOD_THREAT` modifiers + crit `SPELL_AURA_MOD_CRITICAL_THREAT` multiplier + `ApplyTotalThreatModifier` (school-mask scaling), and finally `addThreatDirectly` at line 427 inserts/updates the `HostileReference` in the `ThreatContainer`. Hostile-spell initial threat is gated at [`D:/MaNGOS/source/src/game/Spells/Spell.cpp:1454-1462`](D:/MaNGOS/source/src/game/Spells/Spell.cpp) (the same gate cited by [pull-target.md](./pull-target.md) — non-positive AND not `SPELL_ATTR_EX_NO_THREAT` AND not `SPELL_ATTR_EX2_NO_INITIAL_THREAT` → combat + `addThreat`).

## Overview

Threat is **client-estimated** in 1.12.1. The server tracks per-creature `ThreatContainer` lists with per-`HostileReference` `iThreat` floats (`ThreatManager.h:66` accessor `getThreat()`), but no opcode publishes the table to clients — the 3.x `SMSG_THREAT_UPDATE` / `SMSG_HIGHEST_THREAT_UPDATE` family does not exist yet. The bot therefore observes combat-log events (`SMSG_SPELLNONMELEEDAMAGELOG`, `SMSG_ATTACKERSTATEUPDATE`, `SMSG_SPELLHEALLOG`) and aggregates a local estimate.

The DPS-side decision lives in two places. The **wait-for-threat** gate at [`CombatRotationTask.cs:660-676`](../../../Exports/BotRunner/Tasks/CombatRotationTask.cs) (`ShouldWaitForThreat`): if `IsInGroup AND target.TargetGuid == 0` (mob hasn't picked anyone yet), pause DPS for one tick — this is the "let tank pull first" rule. The **throttle** gate is [`ThreatTracker.ShouldThrottle`](../../../Exports/BotRunner/Combat/ThreatTracker.cs:53-58): once combat is engaged and the tank has aggro, DPS holds at ≤ 90% of tank's running total to leave a 20%-melee / 40%-range cushion before the server flips victim.

**Healer aggro mitigation** is the asymmetric edge case. A heal generates threat equal to half its effective amount, split *across all mobs the heal target is in combat with* on the server side (`SPELL_AURA_MOD_HEALING_PCT` and friends modulate this). The client estimator collapses to "0.5 × heal" added to the healer's running total without the per-mob split — so the healer's estimated threat will diverge from the server when the heal target is engaged with multiple mobs. The mitigation is **tank-pull-first**: tanks should establish ≥ 1000 threat on every mob before healers begin casting; the wait-for-threat gate at line 660-676 enforces this at the rotation level. Healers without a tank in group fall back to "self-heal as low as possible" + bandage between fights, which is the resting flow described in `rest-rotation.md` (planned).
