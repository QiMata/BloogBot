# Spirit Healer — Ghost → Approach → `CMSG_SPIRIT_HEALER_ACTIVATE` → Resurrected (50% HP/mana, 25% durability tax, rez sickness)

Foundational recovery cycle. Sibling to [corpse-run.md](./corpse-run.md): where corpse-run.md describes the *full* death → release → reclaim/spirit-heal/`.unstuck` cycle, this doc isolates the "I accept the durability tax" branch as a first-class Activity surface so the planned `Exports/BotRunner/Tasks/Recovery/SpiritHealerTask.cs` (`not-started` per [Plan/Activities/recovery.md §SpiritHealerTask:392-484](../../Plan/Activities/recovery.md)) has a single authoritative DE/snapshot contract to point at. The shipped behavior today is **inlined** inside `RetrieveCorpseTask.Update()` at [`Exports/BotRunner/Tasks/RetrieveCorpseTask.cs:202-218`](../../../Exports/BotRunner/Tasks/RetrieveCorpseTask.cs) — the `corpseHorizontalDistance > 200f && !_triedSpiritHealer` branch — exactly mirroring the pre-doc state of `ReconnectTask` before iter-8.

The server-side state machine `WorldSession::HandleSpiritHealerActivateOpcode` lives at [`D:/MaNGOS/source/src/game/Handlers/NPCHandler.cpp:433-447`](D:/MaNGOS/source/src/game/Handlers/NPCHandler.cpp); the wire-side response is `SendSpiritResurrect` at [`NPCHandler.cpp:450-497`](D:/MaNGOS/source/src/game/Handlers/NPCHandler.cpp). The 1.12.1 opcode is `CMSG_SPIRIT_HEALER_ACTIVATE = 540 (0x21C)` at [`D:/MaNGOS/source/src/game/Protocol/Opcodes_1_12_1.h:543`](D:/MaNGOS/source/src/game/Protocol/Opcodes_1_12_1.h). The BG-side query opcode `CMSG_AREA_SPIRIT_HEALER_QUERY` (8-byte payload, BG-only) ships at [`Exports/WoWSharpClient/Networking/ClientComponents/DeadActorClientComponent.cs:226-249`](../../../Exports/WoWSharpClient/Networking/ClientComponents/DeadActorClientComponent.cs).

## Overview

A ghost (`PLAYER_FLAGS_GHOST = 0x10`, transformed via `SPELL_AURA_GHOST` spell 8326 + NE-only `20584`) approaches a creature with `UNIT_NPC_FLAG_SPIRITHEALER = 0x20` ([`D:/MaNGOS/source/src/game/Objects/UnitDefines.h:665`](D:/MaNGOS/source/src/game/Objects/UnitDefines.h)) within `INTERACTION_DISTANCE = 5.0f yd` ([`D:/MaNGOS/source/src/game/Objects/ObjectDefines.h:24`](D:/MaNGOS/source/src/game/Objects/ObjectDefines.h)) and sends `CMSG_SPIRIT_HEALER_ACTIVATE` with the healer's `ObjectGuid` as an 8-byte payload. Server-side `HandleSpiritHealerActivateOpcode` resolves the NPC via `Player::GetNPCIfCanInteractWith(guid, UNIT_NPC_FLAG_SPIRITHEALER)` ([`D:/MaNGOS/source/src/game/Objects/Player.cpp:2919-2929`](D:/MaNGOS/source/src/game/Objects/Player.cpp)), interrupts channelling spells, removes `AURA_INTERRUPT_INTERACTING_CANCELS` auras, then drives `SendSpiritResurrect` which performs four observable mutations in order: (1) `ResurrectPlayer(0.5f, true)` ([`Player.cpp:5096-5158`](D:/MaNGOS/source/src/game/Objects/Player.cpp)) — drops `SPELL_AURA_GHOST`, clears NE wisp aura 20584, sets HP/mana/energy to 50% of max, casts `SPELL_ID_PASSIVE_RESURRECTION_SICKNESS = 15007` ([`SharedDefines.h:1170`](D:/MaNGOS/source/src/game/SharedDefines.h)) when `Level >= CONFIG_INT32_DEATH_SICKNESS_LEVEL` (default 11); (2) `DurabilityLossAll(0.25f, true)` ([`NPCHandler.cpp:454`](D:/MaNGOS/source/src/game/Handlers/NPCHandler.cpp)) — 25% durability loss on every equipped slot **including inventory** (the `true` second arg); (3) `SpawnCorpseBones()` — corpse-cleanup so the bot's `Corpse*` is invalidated post-resurrect; (4) `TeleportTo(corpseGrave)` if the corpse-nearest graveyard differs from the ghost-nearest graveyard ([`NPCHandler.cpp:457-490`](D:/MaNGOS/source/src/game/Handlers/NPCHandler.cpp)) — engine MUST re-query position post-resurrect rather than trusting pre-activation coordinates.

The client-side packet path is `IDeadActorNetworkClientComponent.ResurrectWithSpiritHealerAsync(ulong spiritHealerGuid, CancellationToken)` at [`DeadActorClientComponent.cs:178-201`](../../../Exports/WoWSharpClient/Networking/ClientComponents/DeadActorClientComponent.cs) — constructs the 8-byte little-endian GUID payload and `await _worldClient.SendOpcodeAsync(Opcode.CMSG_SPIRIT_HEALER_ACTIVATE, payload, ct)`. The inlined `RetrieveCorpseTask` branch reaches this path **indirectly** through `spiritHealer.Interact()` ([`RetrieveCorpseTask.cs:213`](../../../Exports/BotRunner/Tasks/RetrieveCorpseTask.cs)) which on FG triggers the gossip-menu confirmation and on BG routes through `CMSG_SPIRIT_HEALER_ACTIVATE` — see [Plan/Activities/recovery.md §SpiritHealerTask "BG protocol footprint"](../../Plan/Activities/recovery.md) for the planned task's direct dispatch.

The live test anchor is `BotRunner.Tests.LiveValidation.SpiritHealerTests.SpiritHealer_Resurrect_PlayerAliveWithSickness` at [`Tests/BotRunner.Tests/LiveValidation/SpiritHealerTests.cs:36`](../../../Tests/BotRunner.Tests/LiveValidation/SpiritHealerTests.cs) — same file already cited by [corpse-run.md](./corpse-run.md) Live-test acceptance §"Spirit-heal mode". It exercises the full ghost → approach (`ActionType.Goto` to spirit-healer XYZ + `4.0f` radius) → `ActionType.InteractWith` (LongParam = spirit-healer GUID) → wait for `IsStrictAlive` cycle.

## Failure modes

| Failure | Detection | Recovery |
|---|---|---|
| **No spirit-healer in range** (broad-phase 50 yd returned empty) | `IObjectManager.Units.Where(u => (u.NpcFlags & 0x20) != 0 && u.Position.DistanceTo(player) < 50f).Any() == false` | Pop with reason `NoHealerInRange`. If `RetrieveCorpseTask` is still viable (corpse reachable), defer back to it. If unreachable AND `.unstuck` ready, fall through to corpse-run.md R7 (`.unstuck` dead-path: teleport to ClosestGrave + rez sickness + 1 h CD on spell 20939). | **Faction-gated healer** (R11 violation, server-side silent reject) | Post-`CMSG_SPIRIT_HEALER_ACTIVATE` poll: `PlayerFlags & PLAYER_FLAGS_GHOST` still set after 3 s | Mark this healer GUID as "tried, faction-incompatible" in a per-task blacklist; re-scan `IObjectManager.Units` for next-nearest healer with matching team faction. If none within 50 yd → `NoHealerInRange` (above) |
| **Healer in combat / charmed / dead** (R10 silent reject) | Same as faction-gated: post-send ghost-bit persistence | Same blacklist + re-scan flow. For "healer in combat" specifically, retry after 10 s — the healer may exit combat (rare for graveyard NPCs but possible in contested zones) |
| **Bot ghost-form interrupted by aura before approach** (channelling spell from pre-death that survived release) | Post-arrival snapshot still has channel auras visible | Server's `RemoveAurasWithInterruptFlags(AURA_INTERRUPT_INTERACTING_CANCELS)` ([NPCHandler.cpp:446](D:/MaNGOS/source/src/game/Handlers/NPCHandler.cpp)) clears these on accepted activate — no engine action required.
