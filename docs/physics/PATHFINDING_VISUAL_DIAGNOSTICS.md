# Pathfinding Visual Diagnostics

This is the repeatable loop for turning "the bot got stuck" into a layer-specific
finding. Use it before changing MmapGen, PathfindingService, BotRunner path
execution, or BG movement behavior.

## Stable Exporter

Generate the current reference bundles:

```powershell
$env:WWOW_DATA_DIR = 'D:\MaNGOS\data'
.\tools\scripts\export-pathfinding-reference.ps1 -Route all -Resume
.\tools\scripts\summarize-pathfinding-reference.ps1 -Route all
```

Use `-RefreshRaw` only when you need MmapGen to rebuild raw debug geometry. It is
slow. Normal `-Resume` refreshes the OBJ/CSV views from existing latest data and
does not regenerate tiles unless required.

The exporter now prefers the in-tree `tools\MmapGen\build\MmapGen.exe` when it
exists. Pass `-MmapGenExe .\tools\MmapGen\build\MmapGen.exe` explicitly if you
are comparing against older notes, because the external
`D:\MaNGOS\source\bin\MoveMapGenerator.exe` can produce stale/disconnected probe
tiles.

Outputs are always overwritten in stable locations:

- `tmp/test-runtime/visualization/pathfinding/og-zeppelin/latest/`
- `tmp/test-runtime/visualization/pathfinding/brd/latest/`

Each bundle is categorized:

- `source/` - converted compiled ADT/VMAP/GO bake-input geometry.
- `mmap/` - rendered Detour/MMAP surfaces, including focused crop OBJs.
- `overlays/` - route or reference point overlays.
- `analysis/` - CSV polygon reports.
- `analysis/summary.md` - generated quick-read summary from the latest CSV/logs.
- `analysis/suspicious_poly_*.obj` - tiny OBJs for the worst `zRange`
  polygon(s), generated by the summary script for quick human inspection.
- `logs/` - MmapGen logs proving which tile and GO bake path ran.

Do not create dated one-off OBJ folders for normal iteration. Add a new route to
`export-pathfinding-reference.ps1` and write into `<route>/latest/`.

## Layer Triage

Read the artifacts in this order:

1. **Source geometry**: if the tower, ramp, cave, or WMO is missing or is the
   wrong zone, the bug is tile selection, ADT/VMAP extraction, DBC/GameObject
   spawn loading, or coordinate conversion.
2. **MMAP geometry**: if source is plausible but the navmesh contains a shortcut,
   giant triangle, vertical bridge, missing deck, or cliff-interior platform, the
   bug is in MmapGen/Recast bake rules or walkability filters.
3. **Path query**: if the navmesh is plausible but the chosen corridor is wrong,
   inspect Detour poly refs, off-mesh links, query extents, area costs, and smooth
   path generation.
4. **Path execution**: if the path looks correct but the bot fails while moving,
   inspect BotRunner movement execution through `IObjectManager`. BG and FG should
   receive the same command sequence; BG movement is implemented by the
   `WoWSharpObjectManager` side and must still match FG physics.
5. **Live trace**: pair screenshots with snapshot records. A screenshot without
   player XYZ, current waypoint, velocity/stall state, and current task branch is
   not enough for root cause.

## Polygon Report Red Flags

`MmapVisualize --poly-report` writes one row per rendered polygon after crop and
reachability filters. Inspect:

- `zRange`: large vertical span on a flat deck or platform usually means a
  collapsed/bridging polygon.
- `maxEdge2D`: long edges near ramps, lips, caves, or WMO interiors often show a
  shortcut Recast should not have simplified into one surface.
- `horizontalArea2D`: large area on a small walkway/deck may hide multiple real
  surfaces under one Detour polygon.
- reachable/unreachable split: use as a local-tile clue, not as full-route proof,
  because this simple visualizer does not resolve neighbor tiles.

## Nav Summary Accelerator

Long-route acceleration should not replace detailed mmaps with a lossy "smoothed
mmap." The current scaffold is an optional summary graph beside the detailed
data:

- Graph artifacts live as `*.navsummary.json` files, typically under
  `<WWOW_DATA_DIR>\navsummary\`.
- Enable with `WWOW_ENABLE_NAV_SUMMARY=1` or
  `Navigation:NavSummary:Enabled=true`.
- The service finds a coarse anchor chain, then expands each coarse leg through
  the normal detailed mmap/Detour resolver.
- If any detailed leg returns `no_path`, blocked metadata, or an endpoint snap
  beyond `MaxDetailEndpointDistance`, the summary is rejected and the normal
  detailed query runs.
- Dynamic-overlay requests bypass summaries. The route-result cache key includes
  the graph signature only when summaries are active.

Minimal graph shape:

```json
{
  "schemaVersion": 1,
  "id": "kalimdor-long-routes",
  "mapId": 1,
  "nodes": [
    { "id": "crossroads", "x": -437.0, "y": -2596.0, "z": 96.0 },
    { "id": "orgrimmar-front", "x": 1502.0, "y": -4415.0, "z": 23.0 }
  ],
  "edges": [
    { "from": "crossroads", "to": "orgrimmar-front", "cost": 3200.0, "bidirectional": true }
  ]
}
```

This layer is for performance only. If visual diagnostics show a bad polygon,
hole, bridge, or WMO cliff connection, fix MmapGen/Recast output first.
